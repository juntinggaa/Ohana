# 后台审计 · Product Brief

## 一、问题

每一个家庭都有一个"一直在记住的人"。
她不一定做最多的事，但她一定 **想到了** 所有的事。

唐宁是这样一个人：

- 凌晨 1:37 突然坐起来 —— 因为爸爸的降压药快没了
- 上午十点开会，脑子里同时跑着复诊单、血压表电池、幼儿园手工
- 弟弟回"收到"，看起来事情移交了，但她知道下次还是会回到她

家庭里看到的是"她下单了"。
家庭里没看到的是 **她想到了、追问了、核对了、兜底了**。

## 二、目标用户

| 用户层 | 描述 | 典型场景 |
|---|---|---|
| 主用户 · 夹心层照护者 | 30-45 岁，有 6-12 岁孩子，父母在外地或同城，多个家庭群同时在线 | 唐宁 |
| 决策伙伴 · 伴侣 / 兄弟姐妹 | 愿意帮忙，但只做被明说出来的事 | 周勉、弟弟 |
| 受益人 · 老人 | 不想给孩子添麻烦，常把不舒服说轻 | 唐父唐母 |

中国二三线及一线城市，4-2-1 家庭结构，预计 **3000 万 + 同类用户**（保守估计）。

## 三、解决方案

后台审计是一个 **AI 家庭责任交接系统**，由 3 个 Agent 协同：

1. **Task Capture Agent** —— 从混乱多平台消息中识别真任务
2. **Care Workflow Agent** —— 把一条任务展开成 before / during / after 清单
3. **Accountability Agent** —— 识别模糊承接、缺截止、缺证明、掉回发起人、发起人超载

并配以 **心智负担可视化** —— before/after 切换让评委、家庭成员一眼看到劳动分布。

## 四、为什么是现在

- **LLM 成熟**：DeepSeek、Claude、通义、豆包都能在 2-3 元成本内识别中文家庭消息
- **群消息成为生活基础设施**：家庭群、班级群、物业群同时存在
- **二孩 / 高龄父母叠加**：80 后成为同时照顾两端的一代
- **女性劳动可见化的社会讨论**：从家务到心智负担，正在被广泛讨论
- **AI Agent 的可解释性已可达消费级**：每条建议都能附理由

## 五、竞品 / 替代品分析

| 产品 | 核心 | 不解决的事 |
|---|---|---|
| Cozi / FamilyWall | 家庭日历 | 不识别"谁在记住" |
| 飞书家庭 / 滴答清单 | 个人任务管理 | 不区分发起 / 执行 / 兜底 |
| Hearth Display | 智能家庭日历屏 | 单点显示，无 Agent |
| 微信家庭群 | 沟通工具 | 信息散落，无结构化 |
| 学术研究（Allison Daminger、Eve Rodsky） | 给"心智负担"命名 | 没有可用的消费级产品 |

**真正的市场空白：** 把"想到并安排"作为一类劳动产品化、可量化、可分配。

## 六、差异化

> 普通家庭日历追踪 **谁做了**。
> 后台审计追踪 **谁想到了、谁追问了、谁兜底了**。
>
> 普通提醒 App 问 **"这件事什么时候到期？"**
> 后台审计问 **"这件事归谁？他有没有真正承接？"**

## 七、商业模式（多路径）

1. **C 端订阅 · 家庭计划** —— 49 元/月 含照护者 + 2 名协作家人
2. **B 端 · 雇主关怀福利** —— 互联网公司给 30-45 岁员工提供"照护时间"产品
3. **B 端 · 居家养老 / 社区** —— 给社区养老服务提供后台协同基础设施
4. **保险 / 诊所合作** —— 长期慢病管理（高血压、糖尿病）需要家庭后台
5. **政府 · 家庭照护补贴申报** —— 把后台劳动量化后，作为照护补贴申请凭证

## 八、未来 API 集成

- 微信 / 飞书 / 钉钉群消息接入
- 国家医保平台 · 复诊预约 / 处方提取
- 京东健康 / 美团买药 · 药品自动续单
- iCloud / Google Calendar · 家庭共享日历
- OCR · 处方 / 缴费单识别
- 智能音箱（小爱、天猫精灵）· 老人语音承接

## 九、风险 & 边界

- ⚠️ **不能让 AI 做最终医疗决策** —— 它只准备清单和提醒，下单/吃药/调药由人确认
- ⚠️ **不能给家庭成员"打分排名"** —— 我们衡量的是"分布"不是"考核"
- ⚠️ **隐私** —— 家庭消息高度敏感，需要端到端加密 + 后端代理 + 用户全控数据
- ⚠️ **不能成为另一项唐宁需要维护的系统** —— 体验必须 0 维护
